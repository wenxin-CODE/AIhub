# WebAgent 项目结构分析

## 项目概述

WebAgent 是一个基于 Flask 和 LangChain 构建的智能问答系统，集成了 GLM-4-Flash 大模型 API，支持 PDF 文档处理、向量数据库存储、会话管理和智能搜索功能。该项目旨在为前端 Vue 应用提供智能问答服务，能够根据用户问题提供准确、专业的回答。

## 文件夹架构

```
webAgent/
├── app.py              # Flask 应用主文件，处理 HTTP 请求
├── chat.py             # 会话管理模块，基于 langgraph 框架实现
├── create.py           # 大模型初始化和提示词创建模块
├── search.py           # 搜索功能模块，包括网络搜索和向量数据库搜索
├── tools.py            # 工具函数集合
├── vector.py           # PDF 处理和向量数据库管理模块
├── chroma_db/          # Chroma 向量数据库存储目录
├── rawData/            # 原始 PDF 文档存储目录
├── sessions/           # 会话历史记录存储目录
├── example.pdf         # 示例 PDF 文件
└── README.md           # 项目说明文档
```

## 核心模块划分

### 1. 应用层 (app.py)

- 基于 Flask 框架实现的 Web 服务
- 处理前端 Vue 应用的 HTTP 请求
- 管理客户端会话和 IP 映射
- 提供 API 接口

### 2. 会话管理模块 (chat.py)

- 基于 langgraph 框架实现会话管理
- 集成 GLM-4-Flash 大模型 API
- 管理会话历史记录
- 支持会话持久化

### 3. 大模型接口模块 (create.py)

- 初始化 GLM-4-Flash 大模型
- 创建系统提示词和用户提示词
- 构建搜索增强提示词

### 4. 搜索模块 (search.py)

- 集成 Tavily 网络搜索
- 实现向量数据库搜索
- 支持相似度搜索和最大边际相关性搜索

### 5. 向量数据库模块 (vector.py)

- PDF 文件读取和文本切分
- 文本向量化处理
- Chroma 向量数据库管理
- 支持多 PDF 文件处理

### 6. 工具模块 (tools.py)

- 整合搜索功能
- 提供信息获取接口

## 关键文件功能说明

### app.py

- **功能**：Flask 应用主文件，处理前端请求
- **核心功能**：
  - 提供 `/api/test` 测试接口
  - 提供 `/vueflask` 主要接口，处理前端发送的问题
  - 实现基于 IP 地址的会话管理
  - 集成 SessionManager 处理消息
  - 添加线程安全机制和错误处理

### chat.py

- **功能**：会话管理类实现
- **核心功能**：
  - 基于 langgraph 框架构建状态图
  - 集成 GLM-4-Flash 大模型
  - 管理会话历史记录
  - 支持会话持久化到文件
  - 提供会话创建、消息发送、历史记录获取等方法

### create.py

- **功能**：大模型初始化和提示词创建
- **核心功能**：
  - 初始化 ChatZhipuAI 实例
  - 创建系统提示词和用户提示词
  - 构建包含搜索结果的增强提示词

### search.py

- **功能**：搜索功能实现
- **核心功能**：
  - 集成 Tavily 网络搜索
  - 实现向量数据库搜索
  - 支持返回搜索结果和相似度评分

### vector.py

- **功能**：PDF 处理和向量数据库管理
- **核心功能**：
  - 读取 PDF 文件并切分文本
  - 使用 OllamaEmbeddings 进行文本向量化
  - 将向量化文本存储到 Chroma 数据库
  - 支持从本地加载向量数据库

### tools.py

- **功能**：工具函数集合
- **核心功能**：
  - 整合搜索功能，从向量数据库和网络获取信息
  - 提供 getInfo 函数，返回相关搜索结果

## 技术栈组成

### 后端框架

- **Flask**：轻量级 Web 框架，处理 HTTP 请求
- **Flask-CORS**：处理跨域请求

### 大模型集成

- **LangChain**：大语言模型应用开发框架
- **ChatZhipuAI**：智谱 AI 大模型接口
- **OllamaEmbeddings**：本地文本嵌入模型

### 向量数据库

- **Chroma**：轻量级向量数据库，存储和检索向量数据

### 搜索工具

- **TavilySearch**：网络搜索工具

### 文档处理

- **PyPDFLoader**：PDF 文件读取
- **RecursiveCharacterTextSplitter**：文本切分工具

### 会话管理

- **langgraph**：状态管理和工作流框架
- **JSON**：会话持久化存储格式

### 依赖关系说明

1. **app.py** 依赖于 **chat.py** 提供会话管理功能
2. ~~**app.py** 依赖于 **tools.py** 获取搜索结果~~
3. **chat.py** 依赖于 **create.py** 初始化大模型和创建提示词
4. **chat.py** 依赖于 **tools.py** 获取搜索结果
5. **tools.py** 依赖于 **search.py** 执行搜索操作
6. **search.py** 依赖于 **vector.py** 从向量数据库获取信息
7. **vector.py** 依赖于 **create.py** 中的环境变量配置

## 数据流

1. **请求处理**：前端发送请求到 `/vueflask` 接口
2. **会话管理**：app.py 根据 IP 地址获取或创建会话
3. **信息获取**：调用 tools.py 的 getInfo 函数获取相关信息
4. **向量搜索**：search.py 从 Chroma 向量数据库搜索相关内容
5. **网络搜索**：如果向量数据库无相关结果，使用 Tavily 进行网络搜索
6. **提示词构建**：create.py 根据搜索结果构建增强提示词
7. **大模型调用**：chat.py 调用 GLM-4-Flash 生成回答
8. **会话更新**：更新会话历史并持久化到文件
9. **响应返回**：将回答返回给前端

## 安装和设置

### 环境要求

- Python 3.8+
- Flask
- LangChain
- ChatZhipuAI
- Chroma
- PyPDF2
- python-dotenv

### 安装步骤

1. 克隆项目到本地
2. 创建虚拟环境：`python -m venv venv`
3. 激活虚拟环境：`venv\Scripts\activate` (Windows)
4. 安装依赖：`pip install -r requirements.txt`
5. 配置环境变量：在 `.env` 文件中设置 GML_API_KEY 和 GML_BASE_URL

### 启动服务

```bash
python app.py
```

服务将在 `http://localhost:5000` 上运行

## 核心 API 接口

### 1. 测试接口

- **URL**：`/api/test`
- **方法**：GET
- **功能**：测试服务是否正常运行
- **响应**：`{"message": "Hello from Flask!"}`

### 2. 智能问答接口

- **URL**：`/vueflask`
- **方法**：POST
- **功能**：处理用户问题并返回智能回答
- **请求体**：JSON 格式的问题
- **响应**：智能回答内容

## 会话管理

- 基于 IP 地址的会话隔离
- 会话历史持久化到 `sessions/` 目录
- 支持多轮对话上下文
- 线程安全的会话管理

## 文档处理流程

1. **PDF 读取**：使用 PyPDFLoader 读取 PDF 文件
2. **文本切分**：使用 RecursiveCharacterTextSplitter 切分文本
3. **文本向量化**：使用 OllamaEmbeddings 进行向量化
4. **向量存储**：将向量化文本存储到 Chroma 数据库
5. **相似性搜索**：根据用户问题搜索相关文本片段

## 搜索增强生成 (RAG) 流程

1. **问题分析**：分析用户问题，确定相关领域
2. **信息检索**：从向量数据库和网络获取相关信息
3. **提示词构建**：将检索到的信息整合到提示词中
4. **大模型生成**：调用 GLM-4-Flash 生成回答
5. **结果返回**：将回答返回给用户

## 项目优势

1. **智能会话管理**：基于 IP 地址的会话隔离，支持多轮对话
2. **搜索增强**：结合向量数据库和网络搜索，提供准确信息
3. **模块化设计**：清晰的模块划分，易于维护和扩展
4. **持久化存储**：会话和向量数据持久化，确保数据安全
5. **线程安全**：支持并发请求处理
6. **错误处理**：完善的错误处理机制，提高系统稳定性
